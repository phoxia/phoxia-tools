# Contributing

Read `AGENTS.md` and `.phoxia/project.yaml` before changing the project. Open an issue before adding a substantial tool or dependency.

Run `npm test`, `npm run check` and `npm run build`. Update the changelog when the public version changes. Update documentation and a Feature Record when user-facing behavior or a public contract changes. Architecture decisions belong in an ADR; cross-domain contracts require RFC analysis.

Never commit credentials, personal data, private work or employer material. New dependencies must be license compatible.
